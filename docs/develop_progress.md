SEO 風險控管

換成 Next.js 前台本身不代表 SEO 會下降。只要原本網址、文章內容、標題描述、canonical、sitemap 和圖片路徑都正確保留，原本的 SEO 基礎可以延續。

新版前台也有機會因為網站速度、頁面體驗和版面結構改善，帶來長期 SEO 優勢。

正式切換前，我們會先用測試版檢查所有重要網址，確保首頁、文章頁、分類頁、圖片、購物車、結帳和會員頁都能正常開啟。

目前需要特別確認的是 WordPress sitemap 裡出現的 `/__trashed-8/` 和 `/__trashed-9/`。這兩個網址要決定保留、轉址，或從 sitemap 移除。

成本與取捨

我們目前已經投入的成本包含：

- WordPress 主機 / 會員方案年費
- 網域費用

如果要升級成新版前台，就會增加一筆前端部署成本。以 Vercel Pro 來算：

```txt
US$20 / 月
US$240 / 年
約 NT$7,800 / 年
```

這筆成本換來的是：

- 官網速度更好
- 首頁與版面設計更自由
- SEO URL 可以更完整控管
- 每次改版可以先看測試版
- 有問題可以快速回復上一版
- WordPress 後台仍然可以繼續使用

取捨是：

- 成本會比只用 WordPress 高
- 架構會變成 WordPress + Next.js 兩套系統
- 之後前台設計調整需要透過程式版本控制
- 網站的視覺、速度、SEO 控制和長期擴充性會更好

簡單結論

我們可以保留 WordPress 作為後台，繼續使用現有會員、商品、訂單和金流功能，同時新增一個更快、更有彈性的前台網站。

如果目標是提升品牌形象、網站速度和未來擴充性，Vercel Pro 的一年成本可以視為前台升級的固定基礎成本。
